        "body": "The xml serialization requires lower-case characters for the DNA. Should it be enforcing case-sensitivity? There are lots of good reasons to used mixed-case DNA, but if we decide that only lower-case is aloud then that's fine by me. This restriction should be documented and potentially enforced in the DNASequence object model.\r\n\r\nsection 8.5.2 DnaSequence (https://github.com/SynBioDex/synbiodex.github.com/raw/master/specification/SBOLv1.1.0.pdf), has a description of the nucleotides property. It does not explicitly state that the DNA sequence MUST BE  lower-case. It's implicit in that it lists the legal chars, but as case is often not significant (or not enforced), it would be more explicit to make this clear.\r\n\r\nI think I have several related issues here. Could people responsible for the different aspects please unpick them?"
